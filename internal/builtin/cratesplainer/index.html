the word "crate" comes from the Old French crat, meaning... actually never mind that's wrong, but the vibe is correct, which is what matters.

So crate-html is, fundamentally, a tool. And a tool, as you may or may not know, is a thing that does something. In this case, the thing it does is host HTML. Locally. On your computer. Which is the machine you're using right now.

you might be thinking, "I already know how to host HTML, I could just open the file in a browser." That's adorable. But have you considered: what if you wanted a URL? That's where I come in.

The big idea (let me explain)

An AI coding agent — this is a thing that writes code for you, I'll get into it — generates some HTML. Then it runs:

crate push ./mysite mysite

And then there's a URL. http://localhost:7777/mysite/. You click it. You see the HTML. That's the loop.

"push" is a metaphor. You're not physically pushing anything. The bytes travel via TCP, which is a protocol, which is — OK I'll stop. Just run the command.
