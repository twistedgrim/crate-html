The Commands

There are six. I'll cover all of them. Don't worry if it takes a while.

crate push

This is the big one. The command. If you only learn one, learn this one. (But you should learn all of them. I'll cover all of them. I just said that.)

crate push ./mysite mysite
the first argument is the directory, and the second argument is the name. They're different. The directory contains files; the name does not. I know it can be confusing because they're both words.

What this does, mechanically — and I think this is important to understand — is it walks the directory, creates a tar archive in memory, streams it over HTTP, the daemon extracts it, swaps it into place, and returns a URL.

a "tar" is a Unix archive format from 1979. It stands for "tape archive." Because originally it was for tapes. Like, magnetic tapes. You've probably seen them in movies.

crate ls

"ls" comes from the Unix command "ls", which lists things. So this lists things. Specifically, sites.

$ crate ls
welcome                    1 files   1735 bytes  2026-06-20 13:38
overview                   6 files  21190 bytes  2026-06-20 13:42
cratesplainer              4 files  12977 bytes  2026-06-20 13:55
that last one is you. You're looking at it right now. Crazy, right? You're inside the thing you're learning about. That's called recursion. I can explain recursion if you want.

crate rm

"rm" stands for "remove." This removes things. Sites, specifically.

crate rm old-plan
Note: there is no "are you sure?" prompt. That's not an oversight. That's a deliberate design choice. The thinking is: you typed the name, you meant it. If you didn't mean it, push it again. Storage is cheap. Confirmation prompts are expensive (in attention, not money).

crate open

This opens a site in your browser. Your default browser. The one you set, or the one your operating system chose for you while you weren't looking.

crate open mysite
on macOS this calls open. On Linux it calls xdg-open. On Windows it calls rundll32 url.dll,FileProtocolHandler, which is, frankly, a war crime, but Microsoft made me do it.

crate status

This tells you if the daemon is up. Useful when nothing else is working and you've started to suspect the daemon might not be up.

$ crate status
crated 0.1.0-dev  base=http://localhost:7777  sites=3

If you get a connection-refused error: the daemon is not up. Run crated in another terminal. There. You did it. I'm proud of you.

crate token

This prints the bearer token. Not the URL. Not the config. Just the token. One line. To standard output.

$ crate token
374b48a038727f2581c6fdb77f63d267e5deca16a8d7f1426d837334d042ee76
this exists because before it existed, retrieving the token meant running cat ~/.config/crate/config.yaml | grep token | awk '{print $2}'. Which, if you don't write shell every day, is a small archaeology dig. So now it's a subcommand. You're welcome.
